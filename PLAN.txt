TARGET USERS: Small businesses (medical shops, small hotels, mini supermarkets,
small manufacturers) who can't afford enterprise tools.

================================================================================
                         WHAT WE ARE BUILDING
================================================================================

TWO INTERFACES + ONE BACKEND:

  1. WEB DASHBOARD (Desktop/Laptop)
     - For viewing analytics, forecasts, reports
     - Bulk CSV/Excel upload
     - What-If simulator
     - Download reports

  2. MOBILE WEB APP (Phone Browser — no install needed)
     - Voice input for daily sales/restock logging
     - Camera to snap register photos
     - Quick stock check
     - Alerts & AI suggestions

  3. BACKEND (Python + Streamlit)
     - Gemini AI for data parsing
     - ML models for forecasting
     - SQLite database
     - All logic runs here

================================================================================
                         COMPLETE USER FLOW
================================================================================

STEP 1: ONBOARDING (First time only)
─────────────────────────────────────
User opens the app on phone or laptop.

  User picks business type:
  ┌──────────────┬──────────────┬──────────────┐
  │ Medical Shop │ Restaurant   │ Supermarket  │  ← or "Other"
  └──────────────┴──────────────┴──────────────┘

  User enters initial stock (3 ways):
  ┌───────────────────────────────────────────────────┐
  │ Option A: Voice  → "I have 200 Dolo, 50 Crocin"  │
  │ Option B: Photo  → Snap photo of shelf/register   │
  │ Option C: Upload → Upload existing Excel/CSV file  │
  └───────────────────────────────────────────────────┘

  Gemini AI parses ANY of these into:
  ┌─────────────┬──────────┬────────┐
  │ Item Name   │ Quantity │ Price  │
  ├─────────────┼──────────┼────────┤
  │ Dolo 650    │ 200      │ ₹30    │
  │ Crocin      │ 50       │ ₹25    │
  │ Amoxicillin │ 100      │ ₹85    │
  └─────────────┴──────────┴────────┘

  → Saved to database. Setup done.


STEP 2: DAILY USAGE (Ongoing)
─────────────────────────────
Every day, user just logs what they sold and restocked.

  LOGGING SALES (phone — voice or text):
    User says: "Sold 5 Dolo, 3 Crocin today"
    
    Gemini AI parses:
    → { item: "Dolo 650", qty_sold: 5, date: "2026-02-16" }
    → { item: "Crocin",   qty_sold: 3, date: "2026-02-16" }

    System auto-updates stock:
    → Dolo: 200 - 5 = 195 remaining
    → Crocin: 50 - 3 = 47 remaining

  LOGGING RESTOCK (when supplier delivers):
    User says: "Got 100 Dolo from distributor"
    
    System updates:
    → Dolo: 195 + 100 = 295


STEP 3: AI DOES THE THINKING (Automatic)
─────────────────────────────────────────
As data builds up, AI gets smarter:

  Week 1-2 (14 days of data):
    → Simple averages: "You sell ~20 Dolo/day"
    → Basic stock alerts: "Crocin runs out in 3 days"

  Month 1-2 (30-60 days):
    → Trend detection: "Dolo sales increasing 10%/week"
    → Day-of-week patterns: "Sundays are 2x busier"

  Month 3+ (90+ days):
    → Full ML forecasting (XGBoost + Prophet)
    → Seasonal predictions: "Monsoon = cold medicine ↑ 3x"
    → SHAP explainability: "WHY sales will increase"

  FOR HACKATHON DEMO:
    → Pre-load 6 months of sample data so judges see full power
